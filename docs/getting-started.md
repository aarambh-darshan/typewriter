# 🚀 Getting Started with typewriter

This guide walks you through adding typewriter to your Rust project and generating your first TypeScript and Python types.

---

## Prerequisites

- Rust stable toolchain (1.70+)
- A Cargo project with `serde` for serialization

---

## Installation

Add `typebridge` and `serde` to your `Cargo.toml`:

```toml
[dependencies]
typebridge = "0.3.0"
serde = { version = "1", features = ["derive"] }
```

By default, **TypeScript**, **Python**, **Go**, **Swift**, and **Kotlin** emitters are enabled. To use only one:

```toml
# TypeScript only
typebridge = { version = "0.3.0", default-features = false, features = ["typescript"] }

# Python only
typebridge = { version = "0.3.0", default-features = false, features = ["python"] }

# Go only
typebridge = { version = "0.3.0", default-features = false, features = ["go"] }

# Swift only
typebridge = { version = "0.3.0", default-features = false, features = ["swift"] }

# Kotlin only
typebridge = { version = "0.3.0", default-features = false, features = ["kotlin"] }
```

---

## Your First TypeWriter Struct

```rust
use typebridge::TypeWriter;
use serde::{Serialize, Deserialize};

/// A product in the catalog.
#[derive(Serialize, Deserialize, TypeWriter)]
#[sync_to(typescript, python, go, swift, kotlin)]
pub struct Product {
    pub id: String,
    pub name: String,
    pub price: f64,
    pub in_stock: bool,
    pub description: Option<String>,
    pub tags: Vec<String>,
}
```

### What each line does

| Line | Purpose |
|---|---|
| `use typebridge::TypeWriter;` | Import the derive macro |
| `#[derive(... TypeWriter)]` | Enable type generation for this struct |
| `#[sync_to(typescript, python, go)]` | Tell typewriter **which languages** to generate |
| `/// A product...` | Doc comment — becomes JSDoc in TS, docstring in Python |

## Building

```bash
cargo build
```

typewriter generates files during compilation. You'll see output like:

```
  typewriter: Product → ./generated/typescript/product.ts
  typewriter: Product → ./generated/python/product.py
  typewriter: Product → ./generated/go/product.go
  typewriter: Product → ./generated/swift/Product.swift
  typewriter: Product → ./generated/kotlin/Product.kt
```

## Generated Files

### `./generated/typescript/product.ts`

```typescript
// Auto-generated by typewriter v0.3.0. DO NOT EDIT.

/**
 * A product in the catalog.
 */
export interface Product {
  id: string;
  name: string;
  price: number;
  in_stock: boolean;
  description?: string | undefined;
  tags: string[];
}
```

### `./generated/python/product.py`

```python
# Auto-generated by typewriter v0.3.0. DO NOT EDIT.

from pydantic import BaseModel
from typing import Optional

class Product(BaseModel):
    """A product in the catalog."""
    id: str
    name: str
    price: float
    in_stock: bool
    description: Optional[str] = None
    tags: list[str]
```

### `./generated/go/product.go`

```go
// Code generated by typewriter v0.3.0. DO NOT EDIT.
// Source: Product

package types

// A product in the catalog.
type Product struct {
	Id          string  `json:"id"`
	Name        string  `json:"name"`
	Price       float64 `json:"price"`
	In_stock    bool    `json:"in_stock"`
	Description *string `json:"description,omitempty"`
	Tags        []string `json:"tags"`
}
```

## Adding Enums

typewriter supports both simple and data-carrying enums:

### Simple Enum

```rust
#[derive(Serialize, Deserialize, TypeWriter)]
#[sync_to(typescript, python, go)]
pub enum Category {
    Electronics,
    Clothing,
    Books,
    Food,
}
```

**TypeScript:** `export type Category = "Electronics" | "Clothing" | "Books" | "Food";`

**Python:** `class Category(str, Enum): ELECTRONICS = "Electronics"` ...

**Go:** `type Category string` and equivalent `const` blocks.

### Tagged Enum (Discriminated Union)

```rust
#[derive(Serialize, Deserialize, TypeWriter)]
#[serde(tag = "type")]
#[sync_to(typescript, python, go)]
pub enum OrderEvent {
    Created { order_id: String },
    Shipped { tracking_number: String },
    Delivered,
    Cancelled { reason: String },
}
```

**TypeScript:** Discriminated union with `{ type: "Created"; order_id: string }` etc.

**Python:** Union of BaseModel subclasses with `Literal` discriminator fields.

**Go:** Main Enum `interface{}` and one sub-struct wrapper implementation per variant.

## Generic Structs

typebridge supports generic type parameters:

```rust
#[derive(Serialize, Deserialize, TypeWriter)]
#[sync_to(typescript, python, go)]
pub struct Pagination<T> {
    pub items: Vec<T>,
    pub total: u64,
    pub page: u32,
}
```

**TypeScript:** `export interface Pagination<T> { items: T[]; total: bigint; page: number; }`

**Python:** `class Pagination(BaseModel, Generic[T])` with auto-generated `T = TypeVar("T")`

**Go:** `type Pagination[T any] struct { items []T ... }`

## Cross-File Imports

When a struct references another custom type, imports are auto-generated:

```rust
pub struct UserData {
    pub user: FilterUserDto,
}
```

**TypeScript:** `import type { FilterUserDto } from './filter-user-dto';`

**Python:** `from .filter_user_dto import FilterUserDto`

**Go:** No cross-file imports are generated nor needed! As long as all types are compiled under the same `package_name` directory, Go types are globally visible to the package.

## Customizing Output Directories

Create a `typewriter.toml` at your project root:

```toml
[typescript]
output_dir = "../frontend/src/types"

[python]
output_dir = "../backend/schemas"
```
