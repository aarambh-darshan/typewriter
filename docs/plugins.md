# Plugin System

typewriter supports external language emitter plugins through a dynamic loading architecture. This allows community-contributed emitters to be developed as standalone crates without modifying the core typewriter workspace.

## Architecture

```text
┌──────────────────┐     ┌─────────────────┐     ┌──────────────────┐
│  typewriter-cli  │────▶│  PluginRegistry  │────▶│  .so / .dylib    │
│                  │     │  (load + verify) │     │  (plugin crate)  │
└──────────────────┘     └─────────────────┘     └──────────────────┘
                              │                         │
                              ▼                         ▼
                         EmitterPlugin            TypeMapper impl
                         (trait object)           (language-specific)
```

Plugins are compiled as shared libraries (`cdylib`) and loaded at runtime via `libloading`.

## Plugin Discovery

typewriter discovers plugins from three sources, in order:

1. **Explicit paths** — listed in `typewriter.toml` under `[plugins].paths`
2. **Configured directory** — set via `[plugins].dir` in `typewriter.toml`
3. **Default directory** — `~/.typewriter/plugins/` (automatic)

### Configuration

```toml
# typewriter.toml

[plugins]
# Custom directory to scan for plugin shared libraries
dir = "./my-plugins/"

# Explicit paths to plugin shared libraries
paths = [
    "/opt/typewriter/plugins/libtypewriter_plugin_ruby.so",
]

# Plugin-specific configuration (e.g. for the Ruby plugin)
[ruby]
output_dir = "./generated/ruby"
file_style = "snake_case"
```

## Writing a Plugin

### 1. Create a new Rust crate

```bash
cargo new --lib typewriter-plugin-mylang
```

### 2. Configure `Cargo.toml`

```toml
[package]
name = "typewriter-plugin-mylang"
version = "0.1.0"
edition = "2024"

[lib]
crate-type = ["cdylib", "rlib"]

[dependencies]
typewriter-plugin = "0.1.0"
typewriter-core = "0.5.2"
```

> **Important:** The `crate-type = ["cdylib"]` is required for dynamic loading.

### 3. Implement the plugin

```rust
use typewriter_plugin::prelude::*;

// Implement TypeMapper for your language
struct MyLangMapper;

impl TypeMapper for MyLangMapper {
    fn map_primitive(&self, ty: &PrimitiveType) -> String {
        match ty {
            PrimitiveType::String => "string".to_string(),
            PrimitiveType::Bool => "boolean".to_string(),
            PrimitiveType::U32 | PrimitiveType::I32 => "number".to_string(),
            PrimitiveType::F64 => "float".to_string(),
            _ => "any".to_string(),
        }
    }

    fn map_option(&self, inner: &TypeKind) -> String {
        format!("{}?", self.map_type(inner))
    }

    fn map_vec(&self, inner: &TypeKind) -> String {
        format!("List<{}>", self.map_type(inner))
    }

    fn map_hashmap(&self, key: &TypeKind, value: &TypeKind) -> String {
        format!("Map<{}, {}>", self.map_type(key), self.map_type(value))
    }

    fn map_tuple(&self, elements: &[TypeKind]) -> String {
        let inner: Vec<String> = elements.iter().map(|e| self.map_type(e)).collect();
        format!("({})", inner.join(", "))
    }

    fn map_named(&self, name: &str) -> String {
        name.to_string()
    }

    fn emit_struct(&self, def: &StructDef) -> String {
        // Generate struct output for your language
        todo!()
    }

    fn emit_enum(&self, def: &EnumDef) -> String {
        // Generate enum output for your language
        todo!()
    }

    fn file_header(&self, type_name: &str) -> String {
        format!("// Generated by typewriter. Source: {}\n\n", type_name)
    }

    fn file_extension(&self) -> &str { "mylang" }
    fn file_naming(&self, type_name: &str) -> String {
        to_file_style(type_name, FileStyle::SnakeCase)
    }
}

// Implement EmitterPlugin
struct MyLangPlugin;

impl MyLangPlugin {
    fn new() -> Self { Self }
}

impl EmitterPlugin for MyLangPlugin {
    fn language_id(&self) -> &str { "mylang" }
    fn language_name(&self) -> &str { "My Language" }
    fn version(&self) -> &str { "0.1.0" }
    fn default_output_dir(&self) -> &str { "./generated/mylang" }
    fn file_extension(&self) -> &str { "mylang" }

    fn mapper(&self, _config: &PluginConfig) -> Box<dyn TypeMapper> {
        Box::new(MyLangMapper)
    }
}

// This macro generates the C ABI entry points
declare_plugin!(MyLangPlugin);
```

### 4. Build the plugin

```bash
cargo build --release
# Output: target/release/libtypewriter_plugin_mylang.so (Linux)
# Output: target/release/libtypewriter_plugin_mylang.dylib (macOS)
# Output: target/release/typewriter_plugin_mylang.dll (Windows)
```

### 5. Install the plugin

Copy the shared library to `~/.typewriter/plugins/` or configure its path in `typewriter.toml`.

### 6. Use it

```rust
#[derive(TypeWriter)]
#[sync_to(mylang)]
pub struct User {
    pub name: String,
    pub age: u32,
}
```

```bash
typewriter generate --all
```

## CLI Commands

### `typewriter plugin list`

List all loaded plugins:

```bash
$ typewriter plugin list
Plugins: 3 plugin(s) loaded:

  ● Ruby (Sorbet) v0.1.0
    Language ID:  ruby
    Extension:    .rbi
    Output dir:   ./generated/ruby

  ● PHP v0.1.0
    Language ID:  php
    Extension:    .php
    Output dir:   ./generated/php

  ● Dart/Flutter v0.1.0
    Language ID:  dart
    Extension:    .dart
    Output dir:   ./generated/dart
```

### `typewriter plugin validate <path>`

Validate a plugin shared library before installing:

```bash
$ typewriter plugin validate ./target/release/libtypewriter_plugin_ruby.so
✓ Plugin is valid!
  Name:       Ruby (Sorbet)
  Language:   ruby
  Version:    0.1.0
  Extension:  .rbi
```

### `typewriter plugin info <name>`

Show details about a loaded plugin:

```bash
$ typewriter plugin info ruby
Plugin: Ruby (Sorbet)
  Language ID:  ruby
  Version:      0.1.0
  Extension:    .rbi
  Output dir:   ./generated/ruby
  Loaded from:  /home/user/.typewriter/plugins/libtypewriter_plugin_ruby.so
```

## API Version

Plugins are versioned with `PLUGIN_API_VERSION`. When the `EmitterPlugin` or `TypeMapper` trait changes in a backward-incompatible way, this version is incremented. Plugins built against a different version are rejected at load time.

Current API version: **1**

## Bundled Plugins

typewriter ships with three proof-of-concept plugins:

| Plugin | Language | Extension | Description |
|--------|----------|-----------|-------------|
| `typewriter-plugin-ruby` | Ruby (Sorbet) | `.rbi` | Sorbet type signatures |
| `typewriter-plugin-php` | PHP | `.php` | PHP 8.1+ readonly classes |
| `typewriter-plugin-dart` | Dart/Flutter | `.dart` | json_serializable classes |

## Limitations

- **CLI only.** Plugin-based languages are only supported through the CLI (`typewriter generate`). The proc-macro (`#[derive(TypeWriter)]`) silently skips plugin targets at compile time.
- **No hot reload.** Plugins are loaded once at startup. Changes require restarting the CLI.
- **ABI stability.** Plugins must be rebuilt when `PLUGIN_API_VERSION` changes.
