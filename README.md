## ‚öÝÔ∏è The Problem

Every full-stack team using Rust on the backend faces the same invisible tax: **keeping types in sync across languages.**

```
Backend dev changes:                  Frontend/Mobile dev sees:
‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ                 ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ

    pub id: Uuid,          ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚ñ∫      id: string;
    pub email: String,                  email: string;
    pub name: String,   ‚Üê ADDED        // ‚ùå missing: name
    pub role: Role,                     role: Role;
}                                     }
                                      // Runtime: undefined is not a string
```

OpenAPI codegen? Only works for HTTP APIs. Protobuf? Heavy toolchain. Manual sync? Humans forget.

**Root cause:** Every existing solution uses an *intermediary format* as source of truth instead of the Rust source itself.

---

## ‚úÖ The Solution

**`typewriter`** makes your Rust structs and enums the **single, permanent source of truth** for all your type definitions.

Annotate once ‚Üí generate everywhere. When the Rust type changes, every generated file updates automatically. No schema file. No intermediary format. No drift. Ever.

```rust
use typebridge::TypeWriter;
use serde::{Serialize, Deserialize};

#[derive(Serialize, Deserialize, TypeWriter)]
#[sync_to(typescript, python)]
pub struct UserProfile {
    pub id: Uuid,
    pub email: String,
    pub age: Option<u32>,
    pub role: UserRole,
    pub created_at: DateTime<Utc>,
}

// On cargo build, auto-generates:
// ‚úÖ ./generated/typescript/user-profile.ts
// ‚úÖ ./generated/python/user_profile.py
// ‚úÖ ./generated/go/user_profile.go
```

## üöÄ Quick Start

### 1. Add to Cargo.toml

```toml
[dependencies]
typebridge = "0.1.2"
serde = { version = "1", features = ["derive"] }
```

### 2. Annotate your types

```rust
use typebridge::TypeWriter;
use serde::{Serialize, Deserialize};

/// A user in the system.
#[derive(Serialize, Deserialize, TypeWriter)]
#[sync_to(typescript, python, go)]
pub struct User {
    pub id: String,
    pub email: String,
    pub name: String,
    pub age: Option<u32>,
    pub is_active: bool,
    pub tags: Vec<String>,
}
```

### 3. Build

```bash
cargo build
```

That's it. Check `./generated/typescript/`, `./generated/python/`, and `./generated/go/` for your generated files.

## üéØ Features

### ‚úÖ Phase 1 (Current ‚Äî v0.1.2)

- **TypeScript emitter** ‚Äî `export interface`, `export type` unions, optional fields
- **Python emitter** ‚Äî Pydantic v2 `BaseModel`, `Enum`, `Union` with `Literal` discriminators
- **Go emitter** ‚Äî native `struct` parsing, `interface` data-carrying enums, and `omitempty` optional pointers
- **Generic types** ‚Äî `Pagination<T>` ‚Üí `export interface Pagination<T>` (TS) / `class Pagination(BaseModel, Generic[T])` (Python)
- **Cross-file imports** ‚Äî auto `import type { X } from './file'` (TS) / `from .file import X` (Python)
- **Serde compatibility** ‚Äî auto-reads `#[serde(rename, skip, tag, flatten)]`
- **Custom attributes** ‚Äî `#[tw(skip)]`, `#[tw(rename)]`, `#[tw(optional)]`
- **Doc comments** ‚Äî Rust `///` flows to JSDoc in TS and docstrings in Python
- **Smart type unwrapping** ‚Äî `Box<T>`, `Arc<T>`, `Rc<T>` transparently unwrapped
- **Feature-gated emitters** ‚Äî compile only what you need
- **TOML config** ‚Äî `typewriter.toml` for output directories, file naming styles, readonly mode

## üè∑Ô∏è Attributes

### Serde Compatibility

typewriter **automatically reads `#[serde(...)]` attributes** ‚Äî no need to repeat them:

```rust
#[derive(Serialize, Deserialize, TypeWriter)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum PaymentStatus {
    Pending,
    Completed { transaction_id: String },
    Failed { reason: String, code: u32 },
}
```

### Custom Attributes

```rust
pub struct User {
    #[tw(skip)]                     // exclude from generated output
    pub password_hash: String,

    #[tw(rename = "displayName")]   // override field name
    pub username: String,

    #[tw(optional)]                 // force optional even if not Option<T>
    pub legacy_field: String,
}
```

See the full [Attributes Guide](docs/custom-attributes.md).
