# TypeScript

Generates TypeScript interfaces and optionally Zod validation schemas.

## Type Mappings

| Rust Type | TypeScript Type |
|-----------|-----------------|
| `String` | `string` |
| `bool` | `boolean` |
| `u8`, `u16`, `i8`, `i16` | `number` |
| `u32`, `i32`, `f32`, `f64` | `number` |
| `u64`, `i64`, `u128`, `i128` | `bigint` |
| `Uuid` | `string` |
| `DateTime<Utc>` | `string` |
| `Option<T>` | `T \| undefined` |
| `Vec<T>` | `T[]` |
| `HashMap<K, V>` | `Record<K, V>` |

## Example

**Rust:**
```rust
#[derive(TypeWriter)]
#[sync_to(typescript)]
pub struct User {
    pub id: String,
    pub email: String,
    pub age: Option<u32>,
}
```

**Generated:**
```typescript
export interface User {
  id: string;
  email: string;
  age?: number | undefined;
}
```

## Zod Schemas

Zod schemas are generated by default alongside interfaces. They provide runtime validation:

```typescript
import { z } from 'zod';

export const UserSchema = z.object({
  "id": z.string(),
  "email": z.string(),
  "age": z.number().optional(),
});
```

Disable Zod schema generation:

```toml
[typescript]
zod = false
```

Or per-type:

```rust
#[derive(TypeWriter)]
#[sync_to(typescript)]
#[tw(zod = false)]
pub struct InternalOnly {
    // ...
}
```

## Readonly Fields

Make all fields readonly:

```toml
[typescript]
readonly = true
```

```typescript
export interface User {
  readonly id: string;
  readonly email: string;
}
```

## File Naming

Files use kebab-case by default: `UserProfile` → `user-profile.ts`

Change style in `typewriter.toml`:

```toml
[typescript]
file_style = "snake_case"  # user_profile.ts
# or
file_style = "PascalCase"  # UserProfile.ts
```
